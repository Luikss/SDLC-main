Repositories, Branches & Commits

Repository

The location where your code history is stored.

Branch

"Folder" within the repository containing commits.

Commits

Different stages of your code inside a branch.

Git commands

git init

Initializes or reinitializes a Git repository in the current folder.

git status

Shows the status of your working tree.

git add . git add single_file.txt git add my_folder git add *.rb

Updates the index with the content found in your working tree thus making the content staged for the next commit. An index is like a snapshot of your working tree. When you perform a commit, this snapshot is utilized. Therefore, before doing a commit you should perform git add for any files that were modified or created.

git commit -m "My first commit"

All the staged files will be commited and added to the project history.

git branch

View all branches in your project.

git log

Displays commit logs and allows you to see what changes were made, when, and by whom.

git checkout name

Switch to a different branch and the corresponding version of the project will be loaded automatically for you.

git checkout -b new_name

Create a new branch based on the currently chosen one and also switch to it.

git checkout -- .

Reverts current changes.

git checkout -b new-branch

Switches to a new branch called "new-branch"

git reset --hard 123

Resets to code version 123

git merge

Joins development histories together. In other words, it allows you to take changes from one branch and incorporate it into another branch.

git push

Pushes commited changes to repository.

git pull

Grab changes from the remote repository and apply them locally.
